# 게임 UI/UX 디자인 컴포넌트 라이브러리 제안서

> **Patrick's Game UI/UX Design** — 게임 인터페이스 설계를 위한 인터랙티브 레퍼런스 시스템

---

## 1. 프로젝트 배경 및 목적

### 1.1 배경

게임 UI/UX 디자인은 웹/앱 UI와 달리 **장르별 관행**, **실시간 인터랙션**, **플랫폼(PC/모바일) 차이** 등 고유한 요구사항이 존재합니다. 그러나 게임 UI 전용 레퍼런스 시스템이 부족하여, 디자이너들이 장르별 컴포넌트 설계 시 체계적인 가이드를 참고하기 어려운 상황입니다.

### 1.2 목적

게임 UI/UX 디자이너와 개발자가 **즉시 참조할 수 있는 컴포넌트 라이브러리**를 제공합니다.

- 게임에 특화된 **34개 UI 컴포넌트**의 구조(Anatomy), 디자인 가이드라인, 플랫폼별 설계 지침 제공
- **20개 UX 법칙**이 게임 인터페이스에 어떻게 적용되는지 시각적으로 설명
- 실제로 동작하는 **인터랙티브 데모**를 통해 컴포넌트의 동작 원리를 직접 체험
- **UI/UX 체크리스트**로 게임 인터페이스의 품질을 체계적으로 점검

### 1.3 타겟 사용자

| 사용자 | 활용 방식 |
|--------|----------|
| 게임 UI/UX 디자이너 | 컴포넌트 설계 시 레퍼런스로 참조, Anatomy 다이어그램으로 구조 파악 |
| 게임 개발자 | PC/모바일 플랫폼별 구현 가이드라인 확인, 인터랙션 사양 참고 |
| 게임 기획자 | UX 법칙 기반으로 게임 시스템 설계, 체크리스트로 기획 검증 |
| 포트폴리오 열람자 | 프로젝트 전체를 포트폴리오로 열람 |

| F-03 | 인터랙티브 데모 | 11개 게임 전용 컴포넌트의 실시간 인터랙션 데모 | [prd_demo_interaction](docs/prd_demo_interaction.md) |

### 2.2 기능별 상세 요약

#### F-01. 컴포넌트 시스템

```
34개 컴포넌트
├── 게임 전용 (11개): HUD, 인벤토리, 미니맵, 스킬바, 채팅, 프로그레스바,
│                      버튼, 카드, 다이얼로그, 내비게이션, 툴팁
└── 기본 UI 참조 (23개): 토글, 체크박스, 라디오, 드롭다운, 슬라이더,
                          입력필드, 검색, 탭, 아코디언, 스낵바, 로딩 등
```

- **Anatomy 다이어그램**: 모든 34개 컴포넌트에 번호가 매겨진 SVG 구조도 제공
- **게임 화면 와이어프레임**: 게임 전용 11개 컴포넌트가 실제 게임 화면에서 어디에 배치되는지 시각화
- **PC/모바일 플랫폼 가이드**: 각 플랫폼의 입력 방식, 최소 크기, 인터랙션 차이 안내
- **디자인 가이드라인**: 컴포넌트별 설계 원칙 및 주의사항
- **관련 UX 법칙 연동**: 각 컴포넌트와 관련된 UX 법칙으로 바로 이동 가능

#### F-02. UX 법칙 시스템

```
20개 UX 법칙 (4개 카테고리)
├── 행동 (4): 피츠의 법칙, 도허티 임계값, 피드백 원칙, 목표 기울기 효과
├── 인지 (8): 힉의 법칙, 밀러의 법칙, 심미적 사용성 효과, 테슬러의 법칙 등
├── 기억 (4): 폰 레스토프 효과, 피크-엔드 규칙, 계열 위치 효과, 자이가르닉 효과
└── 경험 (4): 야콥의 법칙, 일관성의 원칙, 재인 vs 회상, 포스텔의 법칙
```

- 각 법칙의 **핵심 원리**, **상세 설명**, **게임 UI 적용 방법** 제공
- **시각적 다이어그램(SVG)**으로 법칙의 개념을 직관적으로 표현
- **관련 컴포넌트 역방향 연동**: 해당 법칙을 적용하는 컴포넌트 목록 표시

#### F-03. 인터랙티브 데모

11개 게임 전용 컴포넌트에 **실제로 동작하는 데모**를 제공합니다.
- 버튼 눌림 효과, 인벤토리 드래그 앤 드롭, 스킬 쿨다운 타이머
- 프로그레스 바 데미지/회복 시뮬레이션, 미니맵 플레이어 이동
- 채팅 메시지 입력, 다이얼로그 확인/취소 인터랙션

#### F-04. UI/UX 체크리스트

7개 카테고리, 37개 항목으로 게임 인터페이스 품질을 점검합니다.
- **정보구조, 레이아웃, 인터랙션, 시각 디자인, 콘텐츠, 접근성, 성능**
- 각 항목에 **필수/권고** 우선순위 표시
- 체크 상태 **로컬 저장** (localStorage)으로 브라우저 재방문 시 유지

## 3. 기술 아키텍처

### 3.1 기술 선택 이유

| 선택 | 이유 |
|------|------|
| **순수 HTML/CSS/JS** | 빌드 도구 불필요, 즉시 배포 가능, 외부 의존성 제로 |
| **Hash 기반 라우팅** | 정적 호스팅에서 SPA 구현 가능, 서버 설정 불필요 |
| **CSS Custom Properties** | 테마 전환을 런타임에서 처리, 추가 CSS 파일 불필요 |
| **Netlify** | 무료, CDN 자동 적용, 간단한 CLI 배포 |

### 3.2 파일 구조

```
GameUXUI/
├── index.html          # 진입점: DOM 구조, 사이드바, 메인 콘텐츠 영역
├── app.js              # 핵심 로직: 라우터, 렌더러, 데모, 헬퍼 (1,425줄)
├── data.js             # 데이터: 34개 컴포넌트, 20개 UX법칙, 체크리스트 (2,572줄)
├── styles.css          # 스타일: 테마, 레이아웃, 컴포넌트, 반응형 (2,440줄)
├── netlify.toml        # 배포 설정
├── docs/               # 기획 문서 (PRD)
│   ├── prd_overview.md
│   ├── prd_components.md
│   ├── prd_ux_laws.md
│   ├── prd_checklist.md
│   └── prd_demo_interaction.md
└── PROGRESS.md         # 작업 현황 & TODO
```
